# Генератор коммерческих предложений

Учебный проект: форма ввода данных о клиенте → выбор шаблона КП → генерация текста → просмотр и редактирование → скачивание PDF или сохранение в список. Сохранённые КП можно просматривать в списке и снова скачивать в PDF.

## Что есть в проекте

| Файл | Назначение |
|------|------------|
| **app.py** | Веб-приложение (Flask): маршруты, проверка ввода, формирование PDF, хранение сохранённых КП в JSON. |
| **proposal.py** | Генерация текста КП: словарь шаблонов `TEMPLATES` и функция `generate_proposal(client, template_id)`. |
| **templates/index.html** | Главная: форма, блок с текстом КП, кнопки «Скачать PDF» и «Сохранить в список», шапка «Сохранённые КП». |
| **templates/list.html** | Список сохранённых КП (название и дата), переход к просмотру. |
| **templates/view.html** | Просмотр одного сохранённого КП и кнопка «Скачать PDF». |
| **requirements.txt** | Зависимости: Flask, reportlab. |
| **saved_proposals.json** | Создаётся при первом сохранении; хранит все сохранённые КП (в .gitignore). |

## Запуск

1. Создайте виртуальное окружение (рекомендуется):
   ```bash
   python -m venv venv
   venv\Scripts\activate
   ```

2. Установите зависимости:
   ```bash
   pip install -r requirements.txt
   ```

3. Запустите приложение:
   ```bash
   python app.py
   ```

4. Откройте в браузере: **http://127.0.0.1:5000**

Дальше: выберите шаблон КП, заполните форму → «Сгенерировать КП» → отредактируйте при необходимости → «Скачать PDF» или «Сохранить в список». Список сохранённых — по ссылке **Сохранённые КП** в шапке (или http://127.0.0.1:5000/saved/).

## Проверки при вводе (крайние случаи)

- Все поля формы обязательны (пустые или только пробелы — ошибка).
- Длина каждого поля — не более 500 символов, длина текста КП при скачивании — не более 100 000 символов.
- Шаблон должен быть выбран из списка (иначе — ошибка).
- При скачивании PDF из текста удаляются управляющие символы, чтобы файл формировался без ошибок.

## Кириллица в PDF

На Windows используется системный шрифт Arial — кириллица в PDF отображается корректно. На Linux/macOS можно положить в папку проекта файл шрифта **DejaVuSans.ttf** (например, из пакета DejaVu Fonts).
